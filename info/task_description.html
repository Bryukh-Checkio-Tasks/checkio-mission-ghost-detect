In the northern waters sailors see Ghost Ship more frequently recent times. Nobody see ships since they met Ghost Ship. But how to distinguish a normal and ghost ship in dense fog? We can use locator for this.

For a normal ship reflected signals have the same durations. So we wait three signals and if they have the same durations, then we are quite. But if you see any difference - it's better to run away.

We have the old discrete locator so test results are looking as sequence of 1 and 0, where each symbol is a value for seconds. 1 is a signal, 0 is pause. So N consecutive 1 is a N seconds signal. Pauses can have various lengths. But our old locator show tests results as a number in decimal form, so you need convert it in binary form before analysis. The binary form of tests have the next pattern r"1+0+1+0+1+" (regexp).

Let's look at examples:
- The number on locator is 21. It's "10101" in the binary form and we see 3 signals with 1 second duration each. This is a normal ship.
- The number is 1587. The binary form - "11000110011 and we see 3 signals with 2 seconds duration but with various pause. It's ok.
- The number is 3687. The binary form - "111001100111 and we see 3 signals. Two signals with 3 seconds duration and one with 2 seconds. It's better to run away.

You are given a number and your mission is recognize is it a normal ship (True) or not.

Input: A number as an integer.

Output: Is it a normal ship or not as a boolean.

Example: 

recognize(21) == True
recognize(1587) == True
recognize(3687) == False

How it is used: This is an example of the simple pattern recognition in texts and discrete signals.

Precondition:
21 ≤ number ≤ 1057222719
